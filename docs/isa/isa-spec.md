# LoongRisc ISA 规范

## 1. 概述

LoongRisc 是面向学习与实验的精简指令集（RISC）架构，目标如下：

- 指令格式规整、易于手工编码和解码实现
- 能覆盖整数程序的基础执行需求
- 支持通过专用指令与异构加速器协同

### 关键参数

- 指令宽度：32 位定长
- 寄存器宽度：32 位
- 地址空间：32 位按字节编址
- 对齐约束：32 位字访问必须 4 字节对齐

## 2. 数据类型与存储

- 基础数据类型：32 位有符号/无符号整数
- 字节序：小端（Little-Endian）
- 立即数字段：
  - `imm16`：16 位立即数（算术/控制流按符号扩展，逻辑按零扩展）
  - `target26`：J 格式 26 位目标字段

## 3. 寄存器与状态

通用寄存器共 16 个（`r0`~`r15`）：

| 寄存器 | 别名 | 说明 |
| --- | --- | --- |
| `r0` | `zero` | 恒为 0，写入无效 |
| `r1` | `ra` | 返回地址别名（可选） |
| `r2` | `sp` | 旧栈指针别名 |
| `r3` | `gp` | 全局指针 |
| `r4-r11` |  | 被调用者保存组 |
| `r12` | `fp` | 帧指针（可选） |
| `r13` | `sp` | 推荐栈指针 |
| `r14` | `lr` | 链接寄存器 |
| `r15` | `pc` | 程序计数器（不可作为普通寄存器直接写） |

### 程序计数器（PC）

- 32 位指令地址
- 仅能由控制流指令修改

### 状态寄存器（SR）

- 仅低 2 位有效：
  - `SR[0]`：`Z`（零标志）
  - `SR[1]`：`N`（负标志）

## 4. 内存模型

- 32 位统一地址空间（代码/数据/MMIO）
- 可选 MMIO 保留区：`0xFFFF0000` ~ `0xFFFFFFFF`
- 基础访存仅定义字读写（`LW`/`SW`）
- 非对齐字访问属于未定义行为

## 5. 指令格式

所有指令均为 32 位定长。

### 5.1 R 格式

`opcode[31:26] rs[25:21] rt[20:16] rd[15:11] funct[5:0]`

### 5.2 I 格式

`opcode[31:26] rs[25:21] rt[20:16] imm16[15:0]`

### 5.3 J 格式

`opcode[31:26] target26[25:0]`

J 目标地址计算：

`PC = (PC & 0xF0000000) | (target26 << 2)`

## 6. 核心 ISA

### 6.1 R 格式算术逻辑

| 指令 | opcode | funct | 语义 |
| --- | --- | --- | --- |
| `ADD rd, rs, rt` | `0x00` | `0x00` | `rd = rs + rt`，更新 `Z/N` |
| `SUB rd, rs, rt` | `0x00` | `0x01` | `rd = rs - rt`，更新 `Z/N` |
| `AND rd, rs, rt` | `0x00` | `0x02` | `rd = rs & rt`，更新 `Z/N` |
| `OR rd, rs, rt` | `0x00` | `0x03` | `rd = rs \| rt`，更新 `Z/N` |
| `XOR rd, rs, rt` | `0x00` | `0x04` | `rd = rs ^ rt`，更新 `Z/N` |
| `SLL rd, rs, rt` | `0x00` | `0x05` | 左移，更新 `Z/N` |
| `SRL rd, rs, rt` | `0x00` | `0x06` | 逻辑右移，更新 `Z/N` |
| `SRA rd, rs, rt` | `0x00` | `0x07` | 算术右移，更新 `Z/N` |

### 6.2 I 格式立即数运算

| 指令 | opcode | 语义 |
| --- | --- | --- |
| `ADDI rt, rs, imm16` | `0x01` | `rt = rs + sext(imm16)`，更新 `Z/N` |
| `ANDI rt, rs, imm16` | `0x02` | `rt = rs & zext(imm16)`，更新 `Z/N` |
| `ORI rt, rs, imm16` | `0x03` | `rt = rs \| zext(imm16)`，更新 `Z/N` |
| `XORI rt, rs, imm16` | `0x04` | `rt = rs ^ zext(imm16)`，更新 `Z/N` |
| `LUI rt, imm16` | `0x05` | `rt = imm16 << 16`，更新 `Z/N` |

### 6.3 访存指令

| 指令 | opcode | 语义 |
| --- | --- | --- |
| `LW rt, imm16(rs)` | `0x06` | 字加载（需对齐） |
| `SW rt, imm16(rs)` | `0x07` | 字存储（需对齐） |

### 6.4 分支与跳转

| 指令 | 格式 | opcode/funct | 语义 |
| --- | --- | --- | --- |
| `BEQ rs, rt, imm16` | I | `0x09` | 相等则 `PC = PC + 4 + (sext(imm16) << 2)` |
| `BNE rs, rt, imm16` | I | `0x0A` | 不等则分支 |
| `BLT rs, rt, imm16` | I | `0x0B` | 有符号小于则分支 |
| `BLE rs, rt, imm16` | I | `0x0C` | 有符号小于等于则分支 |
| `J target26` | J | `0x0D` | 绝对跳转（同 256MB 区域） |
| `JAL target26` | J | `0x0E` | `r14 = PC + 4` 后跳转 |
| `JR rs` | R | `opcode=0x08, funct=0x00` | `PC = rs` |
| `JALR rd, rs` | R | `opcode=0x08, funct=0x01` | `rd = PC + 4`，`PC = rs` |

### 6.5 常见伪指令形式

- `NOP` = `ADDI r0, r0, 0`
- `MOV rd, rs` = `ADD rd, rs, r0`

## 7. 异构加速器扩展

opcode 范围：`0x10` ~ `0x1F`

### 7.1 基础扩展（v0.1）

| 指令 | opcode | 格式 | 语义 |
| --- | --- | --- | --- |
| `ACC_CFG rt, imm16` | `0x10` | I | 用参数块配置 `acc_id=imm16[3:0]` |
| `ACC_START rt, imm16` | `0x11` | I | 非阻塞启动目标加速器 |
| `ACC_WAIT rt, imm16` | `0x12` | I | 阻塞等待目标加速器空闲 |
| `ACC_STAT rt, rd` | `0x13` | R | 粗粒度状态查询（空闲/忙碌/错误） |

### 7.2 增强扩展（v0.2）

| 指令 | opcode | 格式 | 语义 |
| --- | --- | --- | --- |
| `ACC_CFG2 rt, imm16` | `0x14` | I | `imm16` 高 4 位为 kernel，低 4 位为 acc |
| `ACC_LAUNCH rt, imm16` | `0x15` | I | 非阻塞启动指定 kernel |
| `ACC_SYNC rt, imm16` | `0x16` | I | 按掩码等待多加速器 |
| `ACC_BARRIER` | `0x17` | I | 全局等待屏障 |
| `ACC_MOV rd, imm16` | `0x18` | I | 写加速器控制寄存器 |
| `ACC_STAT2 rt, rd` | `0x19` | R | 查询扩展 32 位状态字 |

### 7.3 参数块 descriptor 布局（v0）

`ACC_CFG` / `ACC_CFG2` 中寄存器 `rt` 指向 **字对齐** 的内存地址，该地址处为 **descriptor**：固定头 + 版本号 + 命令类型，后续可扩展载荷而 **无需修改指令 opcode**。

**字节序**：与体系一致，均为 **小端**。

**v0 固定头（16 字节，4 字）**：

| 偏移（字节） | 宽度 | 字段 | 说明 |
| --- | --- | --- | --- |
| `0` | 4 | `magic` | 魔数，固定为 `0x4341524C`（ASCII `"LRAC"` 的小端整型表示，便于识别误用指针） |
| `4` | 4 | `version` | 描述符格式版本；当前文档定义为 **`1`** |
| `8` | 4 | `command` | 命令类型，见下表 |
| `12` | 4 | `payloadWords` | 紧跟在固定头之后的 **载荷长度（以字为单位）**；无载荷时为 `0` |

**`command` 取值（v0）**：

| 值 | 含义 |
| --- | --- |
| `0` | 保留 |
| `1` | **配置并提交参数**：载荷为具体加速器/内核所需的参数区（由运行时与驱动约定，可含输入/输出地址、长度等） |
| `2`~`0xFFFFFFFE` | 实现保留，供未来扩展新命令类 |
| `0xFFFFFFFF` | 非法（不得出现在合法 descriptor 中） |

**载荷区**：从字节偏移 `16` 开始，长度为 `payloadWords × 4` 字节；须保证 **整块 descriptor 仍字对齐**。

**演进规则**：

- 新版本 **只能** 通过增大 `version`、在载荷内增加字段、或新增 `command` 值来扩展；**不得** 改变 `magic` 含义。
- 旧实现遇到 **不支持的 `version` 或 `command`**：行为为 **实现定义**（仿真器应报错或置错误状态；硬件可置错误位）。

### 7.4 同步指令与内存一致性

`ACC_WAIT` / `ACC_SYNC` / `ACC_BARRIER` 除「等待加速器完成」外，若不在规范中说明 **与主存可见性相关的语义**，则软件、仿真器与真实硬件可能对 **输入/输出数据何时对彼此可见** 产生不同理解。本节给出 **两级约定**。

#### 7.4.1 仿真器配置（LoongRisc-Sim，规范性）

适用于当前参考仿真器：**无 cache、无独立 store buffer 模型**，主存为单一字节数组。

在此配置下，下列指令在 **成功返回**（允许继续执行后续指令）时保证：

1. **主机 → 加速器（输入）**  
   在 `ACC_START` / `ACC_LAUNCH` **之前**，CPU 对已映射为加速器输入缓冲的内存区域执行的 **`SW`，对随后启动的加速器可见**（即仿真器视角下，启动时加速器读到的内存内容与 CPU 已写入一致）。

2. **加速器 → 主机（输出）**  
   在 **`ACC_WAIT` / `ACC_SYNC` / `ACC_BARRIER` 返回之后**，CPU 对加速器已写入的输出区域执行的 **`LW` 能读到加速器写入的最终结果**。

3. **多加速器**  
   `ACC_SYNC`（掩码）与 `ACC_BARRIER`：在返回时，上述 **(2)** 对所有被等待的加速器各自产生的输出均成立。

该模型等价于在同步点提供 **acquire-release 强度的全内存序**（无需单独 `fence` 指令即可满足教学仿真）。

#### 7.4.2 目标硬件配置（说明性）

真实 SoC 上常存在 **cache、写合并缓冲、非一致路径** 等。若实现「带缓存的 CPU + 通过内存与加速器共享数据」，**仅靠**「等到加速器空闲」**并不自动** 等价于「所有 store 已到达加速器可见的内存」或「加速器写入已对 CPU 侧 cache 可见」。

目标硬件若声称与本 ISA 兼容，须在实现文档中 **至少** 明确下列之一：

- **方案 A**：在 `ACC_WAIT` / `ACC_SYNC` / `ACC_BARRIER` 边界 **隐含** 所需的主存与 I/O 一致性操作（例如对指定区域的无效化/写回，具体由实现定义）；或
- **方案 B**：不隐含 fence，要求软件在启动前/等待后显式使用 **将来可能定义的 `FENCE` 类指令** 或对 **MMIO 控制寄存器** 的访问序列完成刷写；并在用户文档中列出 **必须与加速器共享的内存区域** 是否必须为 **non-cacheable** 或 **由硬件 snoop**。

**与 MMIO 的关系**：对保留 MMIO 窗口（见第 4 节）的 `LW`/`SW` 是否与普通内存有额外排序要求，由 **平台文档** 定义；本 ISA 不强制其与 `ACC_*` 的合成顺序，除非在平台级说明。

### 7.5 其它说明

- 加速器编号范围 `0`~`15`；`0` 保留为默认加速器。
- `ACC_START` / `ACC_LAUNCH` 为非阻塞，须与 `ACC_WAIT` / `ACC_SYNC` / `ACC_BARRIER` 等配合使用以保证语义完整。
- 参数块除上述 descriptor 头外，**载荷内部** 字段由运行时与具体加速器模型版本约定；变更载荷布局时应 bump `version` 或新增 `command`，避免静默不兼容。

## 8. 一致性约束

以下场景以 ISA 文档为准：

- 仿真器解码与执行行为
- 编译器后端合法指令选择
- 汇编/反汇编编码定义

若多文档出现歧义，以 `docs/isa/isa-spec.md` 和 `docs/isa/encoding-table.md` 为最高优先级。
